# Ralph Progress Log
Started: 2026-02-19
Feature: ProfitPulse Phase 1 - Foundation

## Codebase Patterns
- Tailwind custom tokens in `tailwind.config.ts` under `theme.extend`
- Path alias `@/*` → `./src/*` (tsconfig.json)
- Jest config is `.js` (not `.ts`) — avoids needing ts-node
- Jest setup in `jest.setup.ts` with `setupFilesAfterEnv`
- ESLint strict on unused vars — use eslint-disable comment for stub params
- All UI components in `src/components/ui/`, import via barrel: `import { Button, Input, Card } from "@/components/ui"`
- Components use forwardRef for ref forwarding
- Database types in `src/lib/database.types.ts` — import types from here
- InsForge client in `src/lib/insforge.ts` — `getInsForgeClient()` for browser, `getInsForgeAdmin()` for server
- InsForge SDK requires global `fetch` — tests must mock: `global.fetch = jest.fn()`
- InsForge database uses PostgREST pattern: `client.database.from('table').select().eq()`
- Environment variables template in `.env.example`
- Migration script: `scripts/migrate.ts` — run with `npx tsx scripts/migrate.ts`
- Card with onClick gets role="button", tabIndex, keyboard handlers automatically
- Utility functions in `src/lib/`
- Test files in `__tests__/` directories adjacent to source
